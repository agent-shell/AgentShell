## Integration Tests (Docker openssh-server)

```
docker run -d -p 2222:22 \
  -e PASSWORD_ACCESS=true \
  -e USER_PASSWORD=test \
  -e USER_NAME=testuser \
  linuxserver/openssh-server
```

- connect with password auth → PTY responds to echo
- connect with ed25519 key auth
- wrong password → AuthFailed error reaches frontend
- SFTP: list / upload / download file round-trip
- Zmodem rz: `docker exec` sends lrzsz → app receives file
- Shell integration: PROMPT_COMMAND injected → command captured in FTS5
- pty-input-failed: manually close channel → event reaches frontend

## E2E Tests (Playwright + tauri-driver)

### must-pass flows
1. Profile form → fill host/user/password → Connect → terminal shows shell prompt
2. Type in AI panel → streaming response → CommandProposal card appears
3. Approve safe command → executes in terminal → audit.log entry written
4. Destructive command card → Run button disabled → check box → Run enabled
5. SFTP panel → upload local file → appears in remote dir listing

### regression guards
- Resize window during active session → terminal reflows correctly (no 0×0 PTY)
- Switch between tabs → each terminal renders its own output (no cross-contamination)
- Disconnect during AI stream → all CommandApproval cards cleared
